Start

The process of generating schedules consists of 3 steps:

Step 1: Define participants

In this step you have to create participants and set their availability by selecting time frames in the calendar.

Step 2: Define events

Create events and assing participants to them.

Step 3: Define schedule parameters

Define final parameters for the algorithm and start generating.

priority_highBefore starting there are few things to keep in mind
The algorithm is heavily based on combinatorics. It performs operations such as finding combinations of an n-element set. If you are not familiar with this field of maths, these operations sometimes produce incredibly large numbers, to an extend that they might seem to be fake and unreal. Let's look at the example below:
Let's say we defined 20 events that have to be scheduled one after the other in a row. We want to find out how many ways we can arrange them. How do we do that?
We chose one of all 20 events to be the first one. There are 20 possible choices.
If first element is chosen, there are 19 more elements left. So now we have 19 possibilities to choose the second element. For now we have 20 * 19 possibilities.
We repeat this step until we choose all of the elements. The result is:
And... 20! is equal to... 2 432 902 008 176 640 000 possibilities.
To give you the idea how big this number is: if it was dollars and you lived 70 years, you could spend 1 101 323 292.63 dollars a second for your whole life.
The conclusion is: use this tool wisely :) Some data might generate really big numbers of schedules. And remember that your computer might not have enough computing power to perform some operations.
